How to Use the Annotation Tool

1. Overview

This tool allows you to listen to two versions of a Text-to-Speech (TTS) audio (Male and Female) and evaluate them based on Naturalness, Intelligibility, and Context. You will also mark token-level errors and describe what kind of failure each token shows.

2. Marking Token-Level Errors

Feature: Interactive token error editor

The transcript area inside each panel is interactive. If you hear an error on a specific word or token:

  • Click the token in the panel you are annotating.
  • That token becomes marked as an error and opens a structured error card below.
  • Use the dropdowns in that card to describe the error.
  • Use Remove to delete one token error, or Clear Token Errors to reset the whole panel.
  • The old Incorrect Token Summary box is filled automatically from your token selections.

3. The "Perfect" Button ✨

If an audio clip is distinctively good and has no errors, you can use the ✨ Perfect button located above the form.

Clicking this will automatically:

  • Set Naturalness to "Human-like".
  • Set Intelligibility to "Yes".
  • Set Context to "Not Applicable".
  • Clear any token-level errors and notes.

Use this to speed up annotation for good samples!

4. Evaluation Fields

  • Naturalness: Does the voice sound robotic or like a real human?
  • Intelligibility: Is the audio clear? Can you understand every word?
  • Context: Does the tone match the sentence (e.g., rising tone for questions)? Select "Not Applicable" for simple statements without specific tonal requirements.
  • Error Category: What kind of token-level failure happened. Use the closest label, for example pronunciation/G2P, normalization/symbol reading, code-mix rendering, skipped token, conjunct handling, or prosody/stress.
  • Severity: How serious the token error is in perception. Use "Minor" for small but noticeable issues, "Moderate" for clear errors, "Major" for strong distortions, and "Critical" when the token seriously harms intelligibility or meaning.
  • Subsystem Guess: Your best guess about where the failure came from in the TTS stack. Choose items like text normalization, tokenizer, G2P, prosody model, acoustic model, alignment/duration, vocoder, or "Unknown" if you cannot tell.
  • Annotator Confidence: How sure you are about your diagnosis. Use 1 for low confidence, 3 for medium confidence, and 5 for very high confidence.
  • Number Mistakes (সংখ্যা): List any numbers that were read incorrectly.
  • Conjunct Mistakes (যুক্তাক্ষর): List any issues with complex characters.

5. Saving & Navigation

  • Overall Preference: You MUST select which version (Male or Female) sounded better, or if they were Equal.
  • Save All & Next: Click this button at the bottom to save evaluations for BOTH files and automatically move to the next pending item.
